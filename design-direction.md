# TDev Digital — Design Direction

## Core Positioning

TDev Digital is a modern digital studio building websites, web apps, AI systems and digital infrastructure for ambitious small businesses.

The site should feel:
- premium
- sharp
- composed
- technically credible
- commercially practical
- modern but not gimmicky

It should not feel:
- AI-generated
- generic SaaS
- overly futuristic
- crypto/web3
- childish
- template-based
- overly animated
- like a cheap agency landing page

---

## Visual Direction

The design should feel like a high-end independent digital studio.

Reference feeling:
- editorial portfolio
- technical product studio
- dark architectural website
- premium software consultancy
- modern creative developer portfolio

Avoid:
- massive glowing blue/purple blobs
- random glassmorphism everywhere
- excessive gradients
- generic 3-card service sections
- fake dashboard mockups
- emoji/icons as visual shortcuts
- stock startup illustrations
- overly rounded cards
- neon cyber styling

---

## Layout Principles

Use strong layout rather than decoration.

Preferred:
- generous whitespace
- asymmetric grids
- large confident typography
- restrained motion
- strong vertical rhythm
- sharp section transitions
- fewer but better elements
- large case study previews
- editorial-style project layouts

Avoid:
- every section using the same centred heading + paragraph + card grid pattern
- repeated 3-column cards
- repetitive rounded rectangles
- generic section labels
- decorative noise with no purpose

---

## Typography

Typography should feel confident and premium.

Use:
- large hero type
- tight line height
- clear hierarchy
- short section headings
- precise wording

Avoid:
- generic buzzwords
- long centred paragraphs
- over-explaining
- fake corporate language

Tone:
- direct
- capable
- strategic
- technical
- commercially grounded

---

## Colour

Base:
- near-black
- deep navy-black
- off-white text
- muted blue-grey text

Accent:
- use one main accent only
- blue/cyan is acceptable but must be restrained
- gradients should be subtle and rare

Avoid:
- rainbow gradients
- neon overload
- purple-blue glow everywhere
- high-saturation backgrounds

---

## Components

Cards:
- should not all look the same
- use subtle borders
- avoid heavy shadows
- avoid obvious glassmorphism
- avoid excessive border radius

Buttons:
- sharp, simple, confident
- no oversized pill buttons unless justified
- clear hover states

Images:
- project visuals should dominate the portfolio
- use large screenshots and real work
- do not rely on abstract placeholders forever

---

## Motion

Motion should feel expensive and controlled.

Use:
- subtle reveal on scroll
- small hover movement
- image transitions
- restrained page transitions

Avoid:
- bouncing
- spinning
- excessive parallax
- animation on every element
- random cursor effects at this stage

---

## Homepage Direction

The homepage should not feel like a generic agency website.

Structure:
1. Hero — strong positioning, sparse but memorable
2. Featured Work — large project-led visual section
3. Capabilities — fewer, sharper service explanations
4. How I Work — practical process, not fluffy
5. CTA — direct, calm, confident

The work should carry the design.

---

## Copy Rules

Avoid phrases like:
- "transform your digital presence"
- "unlock your potential"
- "bespoke digital solutions"
- "cutting-edge"
- "seamless experiences"
- "elevate your brand"
- "innovative solutions"
- "empower your business"

Preferred language:
- build
- launch
- improve
- automate
- design
- develop
- systems
- websites
- web apps
- AI tools
- measurable
- practical
- clear
- robust

---

## Build Rules for Cursor

When generating code:
- do not invent random decorative sections
- do not use generic SaaS layouts
- do not overuse cards
- do not add excessive icons
- do not add fake testimonials
- do not use placeholder buzzword copy
- keep components clean and editable
- use real project data where possible
- prioritise layout, spacing and typography before decoration
